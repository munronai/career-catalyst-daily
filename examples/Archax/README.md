The `archax-dotnet-sdk` is a good example of where the AI-generated proposals should not be taken at face-value.

When reviewing the documentation in the `/research/discovery-report.md` I had some questions about the basis of the research. The initial response from Gemini CLI was a rehash of the documentation it produced for the proposal.

With some questioning and manual research, the `data-doubts.md` and `dotnet-selection-query.md` reports were generated (important to note these were not in the original proposal). 

Given the lack of access to good data, this proposal is useful for considering possible challenges for Archax but it is reliant on some guesswork.

Review of the Archax API portal/documentation (not in the proposal) would lead me to simpler proposals that would be faster to implement (e.g. clearer documentation on the JWT)
