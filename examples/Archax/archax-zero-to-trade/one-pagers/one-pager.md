# One-Pager: The Archax "Zero-to-Trade" Sandbox

## The Hook
**Go from "Hello, World" to your first institutional test trade in under 15 minutes.**

## The Problem
Institutional firms are inherently risk-averse. Before committing to the Archax ACE API, their developers need to prove that their systems can handle the Archax lifecycle (auth, scaling, orders, settlement). Currently, getting a "testnet" account is a slow, manual process that creates a "valuation gap" where potential partners lose interest before they even see a trade succeed.

## The Solution
A dedicated **"Zero-to-Trade" Sandbox**—a self-serve, developer-first portal that provides:
1.  **Instant API Credentials:** OAuth-based login (GitHub/SSO) for immediate access.
2.  **Auto-Provisioned Accounts:** Every sandbox user starts with $10M in "simulated USD" and "simulated BTC/ETH."
3.  **Active Order Books:** Synthetic market-making bots provide real-time, dynamic L2 order book data for testing execution algorithms.
4.  **The "API Playground":** An integrated browser-based console (Swagger UI + WS Debugger) to test requests without writing a single line of code.

## The Value
1.  **Sales Acceleration:** Drastically reduce the "technical evaluation" phase of the sales cycle.
2.  **Developer Evangelism:** Build a "community of trust" with institutional developers by providing the best-in-class testing experience.
3.  **Operational Savings:** Automate the high-touch onboarding process, freeing up Technical Account Managers (TAMs) for production-critical work.
4.  **Feedback Loop:** Use sandbox telemetry to identify common developer friction points before they reach production.

## Success Metric
- **The "15-Minute Rule":** 80% of new sandbox users should execute their first successful "limit order" within 15 minutes of login.
